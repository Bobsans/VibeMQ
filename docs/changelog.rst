Планируемые версии
==================

Версия 1.0.0 (в разработке)
---------------------------

**Дата:** Q2 2026

**Новые возможности:**

- Базовая функциональность месседж-брокера
- Pub/Sub с использованием очередей
- Гарантия доставки через подтверждения (ACK)
- Keep-alive (PING/PONG) и автоматические реконнекты
- Поддержка режимов доставки (round-robin, fan-out)
- Аутентификация по токену
- Graceful shutdown
- Health checks для оркестраторов
- Сбор метрик производительности

**Компоненты:**

- ``VibeMQ.Core`` — ядро системы
- ``VibeMQ.Server`` — сервер брокера
- ``VibeMQ.Client`` — клиент для подключения
- ``VibeMQ.Protocol`` — протокол обмена сообщениями
- ``VibeMQ.Health`` — HTTP health check сервер
- ``VibeMQ.Server.DependencyInjection`` — DI интеграция сервера
- ``VibeMQ.Client.DependencyInjection`` — DI интеграция клиента

**Документация:**

- Полная документация на ReadTheDocs
- Примеры использования
- Руководства по развёртыванию

Формат версий
=============

VibeMQ использует `SemVer 2.0 <https://semver.org/lang/ru/>`_ для версионирования:

**Формат:** ``MAJOR.MINOR.PATCH``

- **MAJOR** — несовместимые изменения API
- **MINOR** — новая функциональность (обратная совместимость)
- **PATCH** — исправления багов (обратная совместимость)

**Примеры:**

- ``1.0.0`` — первый стабильный релиз
- ``1.1.0`` — новая функциональность
- ``1.1.1`` — исправление багов
- ``2.0.0`` —breaking changes

Предрелизные версии:

- ``1.0.0-alpha.1`` — альфа версия
- ``1.0.0-beta.1`` — бета версия
- ``1.0.0-rc.1`` — релиз-кандидат

Политика версионирования
========================

Поддержка версий
----------------

- **Текущая версия** — полная поддержка
- **Предыдущая MINOR** — поддержка исправлений безопасности
- **Старые версии** — не поддерживаются

График релизов
--------------

- **MAJOR релизы** — по мере накопления breaking changes
- **MINOR релизы** — ежеквартально
- **PATCH релизы** — по мере необходимости

Уведомления об изменениях
-------------------------

**Breaking changes** анонсируются:

- В changelog
- В release notes на GitHub
- За месяц до выхода новой MAJOR версии

Вклад в проект
==============

Как предложить улучшение
------------------------

1. Создайте issue на GitHub с описанием функции
2. Обсудите с maintainers целесообразность
3. Если одобрено — создайте pull request

Как сообщить о баге
-------------------

1. Проверьте существующие issues
2. Создайте новое issue с описанием:
   - Версия VibeMQ
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Логи и ошибки

Как внести вклад в документацию
-------------------------------

1. Найдите проблему в документации
2. Создайте issue или pull request
3. Опишите предлагаемые изменения
