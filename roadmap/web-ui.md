# Веб-интерфейс управления

**Описание:** Веб-админка для мониторинга и управления брокером.

**Детальный план:**

## Шаг 1: Выбор технологии и архитектура
**Задачи:**
1. Выбрать технологию: Blazor Server или React
2. Определить архитектуру:
   - Отдельный проект для веб-интерфейса
   - REST API для взаимодействия с брокером
   - WebSocket для real-time обновлений

## Шаг 2: Создание проекта
**Файл:** Новый проект `src/VibeMQ.Server.WebUI` (Blazor) или `src/VibeMQ.WebUI` (React)

**Задачи:**
1. Создать проект
2. Настроить базовую структуру
3. Настроить роутинг

## Шаг 3: Реализация API для веб-интерфейса
**Файл:** `src/VibeMQ.Server.Http/Controllers/AdminController.cs`

**Задачи:**
1. Создать Admin API endpoints:
   - `GET /api/admin/queues` - список очередей с метриками
   - `GET /api/admin/queues/{name}` - детали очереди
   - `GET /api/admin/queues/{name}/messages` - сообщения в очереди
   - `POST /api/admin/queues/{name}/purge` - очистка очереди
   - `GET /api/admin/connections` - активные подключения
   - `GET /api/admin/metrics` - метрики
   - `GET /api/admin/dlq` - Dead Letter Queue
2. Аутентификация для admin API

## Шаг 4: Реализация UI компонентов
**Задачи:**
1. Dashboard с общими метриками
2. Страница очередей:
   - Список очередей
   - Детали очереди (размер, подписки)
   - Просмотр сообщений
   - Управление (purge, delete)
3. Страница подключений:
   - Список активных подключений
   - Детали подключения
4. Страница Dead Letter Queue:
   - Просмотр сообщений
   - Retry/Delete операции
5. Страница настроек:
   - Конфигурация брокера
   - Управление пользователями/токенами

## Шаг 5: Real-time обновления
**Задачи:**
1. WebSocket соединение для real-time обновлений
2. Обновление метрик в реальном времени
3. Уведомления о событиях (новые сообщения, подключения)

## Шаг 6: Тестирование
**Задачи:**
1. Тесты UI компонентов
2. E2E тесты
3. Тесты производительности
