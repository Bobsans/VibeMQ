# CI/CD с тестами и публикацией на NuGet

**Описание:** Настроить pipeline для автоматической сборки, запуска тестов и публикации пакетов на NuGet.org.

**Детальный план:**

## Шаг 1: Выбор платформы CI/CD
**Задачи:**
1. Выбрать: GitHub Actions, Azure DevOps, или другой
2. Создать базовый workflow файл

## Шаг 2: Настройка сборки и тестов
**Файл:** `.github/workflows/ci.yml`

**Задачи:**
1. Создать workflow для:
   - Сборки проекта
   - Запуска unit тестов
   - Запуска integration тестов
2. Триггеры: push в main/develop, pull requests
3. Матрица для разных версий .NET (8.0, 10.0)

## Шаг 3: Настройка публикации preview версий
**Файл:** `.github/workflows/preview-release.yml`

**Задачи:**
1. Создать workflow для публикации preview версий
2. Триггер: push в ветку `develop` или создание pre-release tag
3. Версионирование: `1.0.0-preview.{build_number}`
4. Публикация на NuGet.org с `--prerelease` флагом

## Шаг 4: Настройка публикации стабильных версий
**Файл:** `.github/workflows/release.yml`

**Задачи:**
1. Создать workflow для публикации стабильных версий
2. Триггер: создание release tag (например, `v1.0.0`)
3. Извлечение версии из tag
4. Сборка и публикация на NuGet.org
5. Создание GitHub release с changelog

## Шаг 5: Настройка секретов
**Задачи:**
1. Добавить NuGet API key в GitHub Secrets
2. Настроить переменные окружения для CI/CD

## Шаг 6: Тестирование pipeline
**Задачи:**
1. Протестировать сборку и тесты
2. Протестировать публикацию preview версии
3. Протестировать публикацию стабильной версии
